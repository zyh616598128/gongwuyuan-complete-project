# 公务员备考App页面设计方案

## 1. 应用整体设计风格
- **设计语言**: Material Design 3
- **主色调**: 深蓝色 (#1565C0) - 代表专业性和可靠性
- **辅助色**: 绿色 (#4CAF50) - 代表进步和成功
- **强调色**: 橙色 (#FF9800) - 用于重要操作和提醒
- **背景色**: 浅灰色 (#FAFAFA) 或 白色 (#FFFFFF)

## 2. 页面结构设计

### 2.1 启动页 (SplashActivity)
- 应用Logo居中显示
- 应用名称 "公考通"
- 简洁加载动画
- 3秒后自动跳转到主页

### 2.2 登录/注册页 (LoginActivity/MainActivity)
- 顶部显示应用Logo和名称
- 中部：用户名/手机号输入框
- 中部：密码输入框
- 中部：登录按钮（主色调）
- 中部：忘记密码链接
- 底部：注册账号链接
- 底部：第三方登录选项（微信、QQ）

### 2.3 主页 (HomeActivity)
- **顶部工具栏**:
  - 用户头像（左上角）
  - 消息图标（右上角）
  - 搜索图标（右上角）

- **主要内容区**:
  - 横幅轮播图（考试资讯、最新政策）
  - 个人学习进度卡片
    * 今日学习时长
    * 本周学习目标完成度
    * 连续打卡天数
  - 功能快捷入口网格（4x2布局）
    * 顺序练习
    * 随机练习
    * 模拟考试
    * 错题本
    * 章节练习
    * 历年真题
    * 学习计划
    * 视频课程

- **底部导航栏**:
  - 首页（选中状态）
  - 题库
  - 课程
  - 个人中心

### 2.4 题库页 (QuestionBankActivity)
- **顶部搜索栏**:
  - 搜索框（带语音输入）
  - 筛选图标

- **科目分类标签栏**:
  - 行政职业能力测验
  - 申论
  - 公共基础知识
  - 专业知识

- **题目列表**:
  - 使用CardView展示题目卡片
  - 题目标题
  - 题目类型标识
  - 正确率显示
  - 已做次数

### 2.5 练习页 (PracticeActivity)
- **顶部状态栏**:
  - 当前题目序号（X/Y）
  - 计时器
  - 标记题目按钮

- **题目区域**:
  - 题目内容（支持富文本显示）
  - 选项区域（A/B/C/D等）
  - 解析折叠区域

- **底部操作栏**:
  - 上一题/下一题按钮
  - 提交按钮
  - 暂停/继续按钮

### 2.6 模拟考试页 (MockExamActivity)
- **考试信息区域**:
  - 考试名称
  - 题目总数
  - 考试时长

- **题目列表预览**:
  - 显示所有题目编号
  - 已答/未答状态标记
  - 标记题目标识

- **底部操作**:
  - 开始考试按钮
  - 考试说明展开/收起

### 2.7 个人中心页 (ProfileActivity)
- **个人信息区域**:
  - 头像圆形按钮
  - 用户昵称
  - 会员等级/学习天数

- **学习统计区域**:
  - 总题数、已做题数、正确率
  - 学习时长统计图表
  - 正确率趋势图

- **功能列表**:
  - 我的收藏
  - 学习计划
  - 设置
  - 帮助与反馈
  - 关于我们

## 3. 交互设计细节

### 3.1 导航模式
- 底部导航栏：主要功能模块切换
- 顶部工具栏：当前页面操作
- 侧边栏：辅助功能入口

### 3.2 动画效果
- 页面切换使用Material Design过渡动画
- 按钮点击添加水波纹效果
- 列表项添加淡入效果
- 加载状态使用骨架屏

### 3.3 响应式设计
- 适配不同屏幕尺寸（手机、平板）
- 横竖屏切换支持
- 字体大小适配系统设置

## 4. 组件使用建议

### 4.1 布局组件
- **根布局**: CoordinatorLayout
- **主要内容**: NestedScrollView 或 RecyclerView
- **工具栏**: Toolbar 或 MaterialToolbar
- **底部导航**: BottomNavigationView

### 4.2 内容组件
- **卡片**: CardView
- **列表项**: MaterialCardView
- **按钮**: MaterialButton
- **输入框**: TextInputLayout + TextInputEditText

### 4.3 反馈组件
- **轻量提示**: Toast
- **重要提示**: Snackbar
- **确认操作**: MaterialAlertDialog
- **加载状态**: CircularProgressIndicator

## 5. 性能优化考虑
- 使用ViewHolder模式优化列表
- 异步加载图片和数据
- 合理使用内存缓存
- 避免过度绘制

## 6. 可访问性
- 提供足够的颜色对比度
- 支持系统字体大小调整
- 为图片提供内容描述
- 支持键盘导航

这个设计方案遵循了Material Design指南，同时考虑了教育类应用的特点，旨在为用户提供清晰、直观、高效的学习体验。
