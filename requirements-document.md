# Gongwuyuan App 需求文档

## 1. 项目概述

### 1.1 项目名称
公考王 (Gongwuyuan) - 公务员考试备考平台

### 1.2 项目目标
为公务员考试考生提供全面的备考资源，包括题库练习、模拟考试、知识点学习等功能。

### 1.3 目标用户
- 公务员考试考生
- 事业单位考试考生
- 教师资格证考试考生
- 其他相关考试考生

## 2. 系统架构

### 2.1 技术栈
- 后端：Spring Boot + MySQL + Redis + JWT
- 前端：Web端 + Android原生App
- 部署：Docker容器化部署

### 2.2 部署环境
- 服务器：81.70.234.241
- 后端API：http://81.70.234.241:8080
- Web前端：待开发
- 移动端：Android原生应用

## 3. 用户角色

### 3.1 普通用户
- 可以注册/登录
- 可以进行题目练习
- 可以参加模拟考试
- 可以查看个人统计

### 3.2 管理员
- 可以管理题目内容
- 可以管理用户数据
- 可以查看系统统计

## 4. 功能需求

### 4.1 用户管理模块

#### 4.1.1 用户注册
- **功能描述**：新用户可以注册账户
- **输入**：用户名、邮箱、密码、手机号、昵称
- **输出**：注册成功或失败消息
- **验证规则**：
  - 用户名唯一性校验
  - 邮箱格式校验
  - 密码强度校验
  - 手机号格式校验

#### 4.1.2 用户登录
- **功能描述**：已注册用户可以登录系统
- **输入**：用户名/邮箱 + 密码
- **输出**：JWT认证令牌
- **安全要求**：密码加密存储，支持JWT令牌自动刷新

#### 4.1.3 用户资料管理
- **功能描述**：用户可以查看和修改个人信息
- **可查看信息**：用户名、邮箱、昵称、手机号、注册时间、学习天数、答题统计
- **可修改信息**：昵称、手机号

### 4.2 题目管理模块

#### 4.2.1 题目分类浏览
- **功能描述**：用户可以按科目浏览题目
- **科目分类**：
  - 行政职业能力测验
  - 申论
  - 公共基础知识
  - 面试技巧
- **子分类**：各科目下的具体知识点分类

#### 4.2.2 题目练习
- **功能描述**：用户可以选择题目进行练习
- **练习模式**：
  - 按科目练习
  - 按知识点练习
  - 随机组卷练习
- **练习记录**：记录用户的答题历史和正确率

#### 4.2.3 题目详情
- **功能描述**：显示题目的详细信息
- **显示内容**：
  - 题目内容
  - 选项（选择题）
  - 解析
  - 难度等级
  - 知识点标签

### 4.3 考试模块

#### 4.3.1 模拟考试
- **功能描述**：用户可以参加模拟考试
- **考试配置**：
  - 考试时间限制
  - 题目数量
  - 题目类型分布
- **考试监控**：计时器、剩余时间提示

#### 4.3.2 考试结果
- **功能描述**：显示考试结果和分析
- **显示内容**：
  - 总分
  - 正确率
  - 各科目得分
  - 错题列表
  - 学习建议

### 4.4 统计分析模块

#### 4.4.1 个人统计
- **功能描述**：显示用户的个人学习统计数据
- **统计指标**：
  - 总题数
  - 已答题目数
  - 正确率
  - 学习天数
  - 连续学习天数

#### 4.4.2 错题本
- **功能描述**：记录用户的错题
- **功能**：
  - 查看错题
  - 重新练习错题
  - 错题分类整理

### 4.5 学习资源模块

#### 4.5.1 课程资料
- **功能描述**：提供学习资料和课程
- **内容类型**：
  - 视频教程
  - PDF资料
  - 文章解析

#### 4.5.2 学习计划
- **功能描述**：帮助用户制定学习计划
- **功能**：
  - 自定义学习目标
  - 学习进度跟踪
  - 提醒功能

## 5. 非功能性需求

### 5.1 性能要求
- 页面响应时间 < 2秒
- API响应时间 < 500ms
- 支持1000+并发用户

### 5.2 安全要求
- 用户密码加密存储
- JWT令牌安全传输
- 防止SQL注入和XSS攻击
- 敏感数据脱敏处理

### 5.3 可用性要求
- 系统可用性 > 99%
- 数据备份机制
- 故障恢复方案

## 6. API 接口规范

### 6.1 认证接口
```
POST /api/auth/register - 用户注册
POST /api/auth/login - 用户登录
POST /api/auth/logout - 用户登出
```

### 6.2 用户接口
```
GET /api/users/profile - 获取用户资料
PUT /api/users/profile - 更新用户资料
```

### 6.3 题目接口
```
GET /api/subjects - 获取科目列表
GET /api/questions - 获取题目列表
GET /api/questions/{id} - 获取题目详情
GET /api/questions/subject/{subjectId} - 按科目获取题目
```

### 6.4 考试接口
```
POST /api/exams - 创建考试
GET /api/exams/history - 获取考试历史
POST /api/exams/{id}/submit - 提交考试
```
